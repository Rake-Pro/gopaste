UI Preview // v1

Frontend design for gopaste in the RAKEPRO brand system - tactical/UNSC navy, steel-blue accents, tactical-green for active state. A full-bleed editor with a line-number gutter and an edit/view duality, fronted by a labeled HUD command bar and a status bar that surfaces language, line count, key and expiry. All color comes from --tokens on a [data-theme] root, so this "rake" base theme is swappable later.

What changes, what stays
  • KEEPFull-bleed editor + line-number gutter + edit/view duality. Familiar pastebin muscle memory; fast to read and write.
  • CHANGEHUD command bar with labeled actions replaces the floating top-left icon cluster whose meaning you had to hover to learn. Same actions, legible, with visible shortcuts.
  • NEWStatus bar: mode (NEW/VIEW), key, detected language, line/char count, and expiry countdown.
  • NEWCopy link + copy contents via the Clipboard API, with a toast. The old flow made you select the URL by hand.
  • DROPTwitter button (dated, and pulls a third party). Replaced by Copy link; a generic Share can return later.
  • DROPjQuery 1.7.1 + Google CDN + solarized theme. Move to small vanilla JS, self-hosted, embedded - matches the zero-external-dep posture of the Go backend.
  • CHANGETactical syntax theme derived from brand tokens (steel/green/cyan) instead of solarized, so code and chrome share one palette.
  • NEWTheme system: every color is a token on [data-theme]. "rake" ships first; light / solarized / custom themes are drop-in token blocks with a switcher in the status bar.
